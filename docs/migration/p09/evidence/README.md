# P09 证据目录

`runs/` 是统一验证的唯一运行证据位置。每一次执行必须创建新的目录；已有目录、符号链接或缺失哈希的目录不构成完成证据。

运行目录至少包含：

- `summary.json`：阶段状态、命令时间线、实际退出码、脱敏日志引用、输入/源文件哈希、owner 时间线、资源行数/表哈希、清理和剩余风险；
- `evidence-manifest.json`：除 manifest 自身外每个证据文件的字节数与 SHA-256，并标记运行目录不可覆盖；
- 每条命令独立的 stdout/stderr 脱敏日志；
- 临时规模副本 manifest 与切换/恢复演练报告的安全副本（若相应步骤已经成功）。

证据采集拒绝秘密、token、cookie、session、真实用户数据、用户绝对路径、file URL、生产数据库路径、符号链接和超过大小上限的文件。失败/阻断运行仍保留已执行步骤的真实退出码和安全日志，但绝不补写“成功”结果，也不会继续执行尚未授权的切换步骤。

根目录的 `manifest.json` 只是证据格式与风险声明，不是一次验证成功的替代品。实际运行证据必须来自 `runs/` 中通过哈希检查的不可变目录。

# P15 降级与安全停止手册

## 立即停止

出现未知 IPC、Node/sql.js 打开或写入、两个 writer、会话泄漏、非允许 Origin、sidecar readiness 异常、包内 binary 不匹配、签名失败、迁移完整性失败或证据含秘密/未授权路径时：停止当前临时 run，关闭 sidecar 与其进程树，撤销会话材料，并将证据状态设为 `blocked`。

## 不允许的降级

- 不回退到 Node/sql.js 打开已由 Go 接管的数据库。
- 不让 Node 接管 Go 会话、Terminal、Chat、Loop、脚本或执行器。
- 不把未签名/未公证包当作发布候选，不绕过平台缺失。
- 不对真实 userData 自动执行迁移、恢复、降级或清理。

## 允许的恢复边界

在隔离目录中保留失败摘要、命令退出码、二进制/备份哈希和稳定错误码。将应用维持在不可写或退出状态；修复后从新的临时 run 重新验证。对于已写入的生产数据，只允许兼容 Go 版本、前向修复或经验证的 Go 恢复流程。

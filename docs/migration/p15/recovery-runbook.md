# P15 恢复手册

本手册仅适用于经过授权的数据库副本和恢复包。真实 userData 必须由人工在独立变更流程中确认；P15 自动化不得触碰它。

1. 停止 Electron、Go sidecar 和所有受其监督的进程树，确认没有活动数据库 owner lock。
2. 对恢复包的 manifest、SQLite 文件、WAL/SHM 状态、来源版本、目标兼容 Go 版本和 SHA-256 进行校验；任何不匹配均停止。
3. 仅将授权副本恢复到隔离临时目录，运行迁移前表计数/关联检查、迁移、`PRAGMA integrity_check`、附件和工作区路径解析、迁移后计数/孤儿审计。
4. 使用匹配的 Go sidecar 启动受控会话，验证 readiness、唯一 writer、核心读取和安全关闭。Node/sql.js 不得参与恢复后的数据库打开或写入。
5. 将实际命令、退出码、时间、平台、源修订、备份/结果哈希和脱敏日志摘要交给证据采集器。失败时删除临时副本，不发布、不切换真实数据。

恢复完成不是发布授权；仍须满足发布手册的三平台、签名和人工批准门禁。

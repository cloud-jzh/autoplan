# 反馈 #36 开发计划：切换项目后反馈输入框无法点击输入

## 背景与判断

反馈现象：在工作区切换项目后，进入或停留在「反馈」模块时，反馈输入框无法直接点击输入；需要切换一次 Windows 窗口，让应用失去焦点再重新激活后才能输入。

基于当前代码分析，反馈输入框由 `IntakePanel` 复用 `Composer` 渲染，草稿由 `useComposerDrafts(projectId)` 按项目隔离；项目切换由 `WorkspaceSidebar` 的原生 `<select className="project-select">` 触发 `useWorkspaceController.switchProject` 后 `navigate` 到新项目路由；快照由 `useSnapshot(projectId)` 异步加载。输入框本身没有禁用逻辑，因此更可能是项目切换期间原生 select 焦点残留、旧项目快照短暂复用、或 Composer 在项目切换后缺少明确的可聚焦边界共同导致的 Windows/Electron 焦点失活问题。

本次修复应限定在渲染层，不改 IPC、数据库和业务执行逻辑。

## 任务拆解

- [x] P001: 项目切换控件释放原生焦点 <!-- scope: src/renderer/components/workspace/WorkspaceSidebar.tsx -->
  - 验收要点：`project-select` 在 `onChange` 触发项目切换前主动释放当前原生控件焦点，避免 Windows/Electron 原生下拉控件在路由切换后继续占用输入焦点。
  - 验收要点：鼠标切换项目、键盘切换项目、返回项目列表、切换深浅色主题、侧边栏对话列表等既有交互不受影响。
  - 验收要点：项目切换仍只调用既有 `onSwitchProject(Number(event.target.value))` 路径，不引入新的项目状态来源。

- [x] P002: 隔离项目切换期间的旧快照渲染 <!-- scope: src/renderer/hooks/useSnapshot.ts,src/renderer/pages/WorkspacePage.tsx -->
  - 验收要点：`projectId` 变化后，工作区不再继续把上一项目的完整 `snapshot` 当作当前项目内容渲染；新项目快照到达前显示加载态或仅保留安全的项目列表信息。
  - 验收要点：`WorkspacePage` 对 `snapshot.activeProjectId` 与路由 `projectId` 做一致性保护，避免旧项目的反馈 Composer 在新路由下短暂挂载并接收点击。
  - 验收要点：已有搜索、Plan 预览、任务筛选、验收视图、脚本视图在项目切换后的状态清理逻辑保持一致，不出现跨项目内容串扰。

- [x] P003: 稳定需求与反馈 Composer 的可聚焦边界 <!-- scope: src/renderer/components/Composer.tsx,src/renderer/components/IntakePanel.tsx,src/renderer/pages/WorkspacePage.tsx -->
  - 验收要点：`Composer` 能接收项目与 intake 类型相关的稳定身份键，项目或类型变化后重置内部 transient 状态，并确保新 textarea 是当前可交互节点。
  - 验收要点：切换到新项目的「反馈」模块后，第一次点击反馈 textarea 即可获得焦点并输入中文、英文和换行内容，不需要让应用窗口失焦再激活。
  - 验收要点：需求输入框与反馈输入框继续分别绑定 `composerDrafts.requirement` / `composerDrafts.feedback`，草稿仍按项目隔离保存，不互相覆盖。
  - 验收要点：不做无条件自动聚焦抢占；用户停留在任务、设置、聊天等非 intake 页时，不应被 Composer 抢走焦点。

# 优化需求/反馈计划生成提示词的 skill 传递规则

## 需求概要

反馈 #55 要求在需求、反馈生成计划的提示词中增加明确约束：当用户在需求或反馈中指定某个 skill 时，计划生成器不能把“使用某 skill”拆成独立任务项，而应把该 skill 使用要求传递到真正需要执行的具体开发任务中。结合现有代码，影响范围集中在计划生成 prompt 组装逻辑，需覆盖外部 CLI Markdown、外部 CLI 结构化 PlanSpec、内置 LLM 结构化 PlanSpec，以及长需求分阶段计划生成路径，并补充 prompt 捕获回归测试，确保该约束与 AutoPlan 任务拆解硬格式、PlanSpec 契约和项目级 Prompt 约束同时存在。

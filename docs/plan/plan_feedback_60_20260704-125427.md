# 反馈 #60 开发计划：修复工作区终端 xterm 依赖解析失败

## 反馈概述

Vite 在加载 `src/renderer/components/workspace/WorkspaceTerminalView.tsx` 时提示：

`[plugin:vite:import-analysis] Failed to resolve import "@xterm/xterm" from "src/renderer/components/workspace/WorkspaceTerminalView.tsx"`

该错误发生在终端视图的 xterm.js 引入位置：

- `import { FitAddon } from '@xterm/addon-fit';`
- `import { Terminal as XTerm } from '@xterm/xterm';`
- `import '@xterm/xterm/css/xterm.css';`

## 现状分析

1. `package.json` 已在 `dependencies` 中声明 `@xterm/addon-fit` 和 `@xterm/xterm`，说明终端视图预期依赖真实 xterm.js 渲染，而不是临时 mock 或移除功能。
2. `package-lock.json` 已包含 `node_modules/@xterm/addon-fit` 与 `node_modules/@xterm/xterm` 条目，版本分别为 `0.11.0` 与 `6.0.0`。
3. `vite.config.ts` 的 `optimizeDeps.include` 已包含 `@xterm/xterm` 和 `@xterm/addon-fit`，当前报错不像是缺少 Vite 预构建配置，而是 Vite 无法在实际安装树中解析包。
4. 当前工作区 `node_modules/@xterm` 目录不存在，`npm ls @xterm/xterm @xterm/addon-fit --depth=0` 返回空依赖树，与 Vite 的 import-analysis 报错一致。
5. `WorkspaceTerminalView.test.tsx` 已通过源码断言覆盖 xterm 导入和终端写入、resize、replay 等契约，但这种测试不会验证依赖是否真实安装；最终必须通过干净安装和 `vite build`/`npm run build` 捕获解析问题。

## 修复策略

优先修复依赖安装树与清单/锁文件的一致性，确保从干净 checkout 执行安装后一定生成 `node_modules/@xterm/xterm` 与 `node_modules/@xterm/addon-fit`。在此基础上保留 `WorkspaceTerminalView` 的真实 xterm 渲染链路，只做最小必要调整，不通过关闭 Vite overlay、删除终端页面或替换为假终端来规避问题。

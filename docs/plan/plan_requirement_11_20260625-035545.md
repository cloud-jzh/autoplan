# 需求 #11 开发计划：Plan 支持全文 Markdown 阅读

## 背景

当前 AutoPlan 工作区的“任务与计划”区域中，Plan 列表仅展示文件路径、任务完成数、耗时、校验状态、哈希和更新时间等摘要信息；计划草稿虽然可以在 textarea 中查看和编辑原始 Markdown，但已确认加入任务系统的正式 Plan 缺少“阅读全文”的入口。随着计划内容变长，用户无法直接在应用内阅读完整计划，也无法按 Markdown 结构查看标题、列表、任务勾选项、代码块和链接等内容。

需求 #11 要求“plan 计划，要支持阅读全文；用 markdown 组件阅读”。本计划默认实现为：在当前项目工作区的正式 Plan 列表中提供“阅读全文”入口，按需从工作区内的 Plan 文件读取 Markdown 全文，并通过专用 Markdown 阅读组件渲染；同时保留现有 Plan 摘要、任务执行、计划草稿编辑和搜索/过滤行为。

## 总体验收标准

- 全文可读：用户能从工作区正式 Plan 列表打开任意当前项目 Plan 的完整 Markdown 正文，而不是只能看到摘要信息。
- Markdown 渲染：正文通过专用 Markdown 组件渲染，标题、列表、任务勾选项、代码块、表格、引用和链接具备清晰可读的视觉样式。
- 按需加载：Plan 全文通过用户点击后按需读取，不随 `snapshot()` 高频刷新批量传输长文本。
- 安全隔离：读取文件仅限当前项目工作区内数据库记录对应的 Plan 文件，路径越界、文件缺失和读取失败均有明确错误提示。
- 流程无回归：计划草稿编辑、确认加入任务系统、任务执行/停止、事件流刷新、搜索/过滤和项目切换保持原有行为。
- 可访问性可用：阅读全文入口和阅读面板支持键盘访问、可关闭、可重试，窄屏下长文、表格和代码块不破坏布局。
- 验证通过：完成实现后 `npm run check` 与 `npm run build` 均通过，关键读取场景和手动阅读场景验证通过。

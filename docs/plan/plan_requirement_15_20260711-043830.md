# P08：剩余静态持久化、Secrets 抽象与稳定 API

## 需求概要

在 P07 已有真实通过证据且 schema、数据库 owner、显式副本和安全门禁均满足的前提下，依据 D:\docs\autoplan-go-backend-separation-plan.md v1.0，将 scripts/executors 元数据、conversations/chat_messages、ai_configs、claude_cli_configs 与实际 MCP 配置迁入本机 Go sidecar 的 domain/repository/application 分层，保持现有 DTO、snake_case、字段存在性、null、排序、项目归属、AppSnapshot 和业务错误兼容，不顺手重写 UI。本阶段只提供静态 CRUD、分页、关联、事务与 snapshot：scripts/executors 的 run/stop/action/import-and-run、Chat send/stop/queue/stream、MCP start/stop/listener 等运行时能力默认安全关闭，并以稳定非 2xx NOT_IMPLEMENTED/能力关闭错误响应，绝不启动外部进程、建立 Chat 流、写运行态或伪造成功。HTTP、MCP 和 UI transport 复用同一 application service；秘密只以 secret_refs 进入业务库，平台 secrets 优先 OS 凭据库、测试可注入 fake，无可用凭据库时使用安装级密钥与受认证加密 fallback。旧明文秘密仅在脱敏 fixture、系统临时库或调用方明确副本上演练迁移、恢复和不可逆清理，Node/sql.js 与 Go 绝不同写同库，也不自动发现或操作真实 Electron userData。任何 P07/P00 基线、安全或副本前置失败均标记 blocked 并停止；交付代码、文档、自动测试、真实命令及退出码、脱敏证据、文件清单和剩余风险。

# P10：统一异步 Operation 与可恢复 SSE 事件协议

## 需求概要

在 P09 代码与数据库副本切换演练门禁通过后，依据 D:\docs\autoplan-go-backend-separation-plan.md v1.0，在脱敏 fixture、临时库或调用方明确副本上建立供后续 Loop、计划生成、任务、脚本、执行器与 Chat 长任务复用的 Operation、事务 outbox 和实时事件基础设施。阶段实现 queued/running/succeeded/failed/cancelled/interrupted 状态机、请求幂等与稳定冲突、确定性取消竞争、每项目 revision、可清理重放的递增事件、鉴权与 Origin 一致的项目/Operation SSE，以及 renderer 重连、去重、缺口检测、合批和 snapshot resync；REST、MCP 与 UI adapter 统一调用 application service。保持既有 DTO、snake_case、snapshot 和业务规则，不重写 UI，不迁移具体长任务执行器，不切换或自动访问真实 Electron userData，并确保 stdout/stderr 有界、截断、脱敏且终端原始输出不进入项目 SSE。交付自动测试、协议与恢复文档、真实命令/退出码、脱敏证据、文件清单和风险；P09 或安全门禁失败时必须 blocked，P00 既有红灯不得新增且 P10 专项测试必须通过。

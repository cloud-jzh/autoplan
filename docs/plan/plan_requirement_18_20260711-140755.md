# P11：Go 核心调度、计划任务动作与 Agent CLI 运行时迁移

## 需求概要

在 P10 的统一 Operation、事务型 event_outbox、可恢复 event bus 与 SSE 已通过的前提下，依据 D:\docs\autoplan-go-backend-separation-plan.md v1.0（2026-07-11），把每项目 Loop 调度、计划/任务/验收动作、失败重试与恢复，以及 Codex、Claude、OpenCode、oh-my-pi Agent CLI 的启动和取消迁入 Go。实现每项目串行 actor/命令队列与全局有界 worker pool，以可注入 fake clock/process/event bus 验证并发和故障；所有外部进程统一经过参数数组、context、超时、输出上限/脱敏、P05 Files workdir policy 和跨平台进程树清理。REST、MCP 与 UI 复用同一 application service，长动作统一返回 P10 Operation 并经 SSE 报告状态。HttpAutoplanClient 按 Loop、Plan、Task、Acceptance/Retry 和 Agent CLI 能力独立切换；Go 路径稳定前保留 Node runtime 的显式回退，但 Node 只能经 GoDataClient 读写数据，禁止自动降级导致重复执行、禁止恢复 Node/sql.js 数据库 owner。全过程只使用脱敏 fixture、系统临时库或明确副本，不自动操作真实 Electron userData，保持既有 DTO、snake_case、snapshot、provider/config/会话、状态机和业务规则，不顺手重写 UI；P10 或安全门禁失败时阶段必须 blocked。

# P12：脚本与执行器统一 Operation / Process Runner 迁移

## 需求概要

在 P11 的 Go 核心调度、统一 Process Runner、跨平台进程树清理和 Agent CLI 运行时已经通过的前提下，依据 D:\docs\autoplan-go-backend-separation-plan.md v1.0（2026-07-11），把脚本的手动、Hook、定时运行/停止以及 shell、process、plugin 执行器的依赖编排、生命周期动作、停止和日志迁入 Go 模块化单体。所有入口复用 P10 Operation/event_outbox/SSE 与 P11 runner，只能按项目归属加载已持久化定义，以参数数组启动，经 P05 Files policy 校验脚本源文件和 workdir，并通过受控 secret provider 注入秘密；禁止请求携带任意 command、shell、cwd、env 或 manual 等字段绕过授权。实现每项目与全局并发、超时、输入输出和日志上限，确定性取消竞争、重启 running→interrupted、跨平台进程树清理、截断标志与审计事件。REST、现有 MCP 工具和 UI adapter 共用 application service；renderer 通过 REST/SSE 按 go_scripts_api、go_executors_api 独立切换，旧 Node runtime 回退只能经 GoDataClient 访问 Go owner 数据，且进程始终由创建它的 runtime 管理到终态。保持既有 DTO、snake_case、snapshot、脚本 Hook/fail_aborts、执行器依赖/plugin action 和业务规则；只使用脱敏 fixture、临时库或调用方明确副本，不自动发现或触碰真实 Electron userData，不允许 Node/sql.js 与 Go 双写。交付代码、文档、自动测试、真实命令与退出码、脱敏证据、文件清单和剩余风险；P11 或安全门禁失败时必须 blocked，P00 既有红灯不得新增且 P12 专项测试必须通过。

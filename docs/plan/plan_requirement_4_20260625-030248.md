# 需求 #4 开发计划：任务系统增加耗时统计

## 背景

当前任务系统已支持 Plan 解析、任务状态流转、并发执行、手动执行/停止、任务列表展示和 smoke 回归。需求 #4 要求在任务系统中增加耗时统计，核心目标是让每个任务从开始执行到结束/停止/失败的耗时可被记录、查询和展示，并在不影响既有任务执行流程的前提下兼容历史数据。

## 任务拆解

- [x] P001: 扩展任务耗时数据模型 <!-- scope: src/database.js,src/renderer/types.ts -->
  - 验收要点：`plan_tasks` 表新增任务开始时间、结束时间、累计耗时毫秒等字段，并通过现有迁移机制兼容旧数据库。
  - 验收要点：历史任务在迁移后字段有安全默认值，空值或 0 值不会影响任务列表、Plan 进度和验收流程。
  - 验收要点：前端 `PlanTask` 类型同步新增耗时字段，TypeScript 编译不出现类型缺口。

- [x] P002: 在任务生命周期记录耗时 <!-- scope: src/loopService.js -->
  - 验收要点：任务进入 `running` 时记录本次 `started_at`，手动执行、循环执行和并发执行路径均覆盖。
  - 验收要点：任务完成、失败回退、手动停止时计算本次耗时并累加到任务耗时字段，同时写入 `finished_at` 和 `updated_at`。
  - 验收要点：任务重试时不丢失历史累计耗时，运行中实时耗时可通过 `started_at` 推导，状态流转保持原有语义。
  - 验收要点：`task.executed`、`task.failed` 或停止相关事件的元信息包含耗时，便于后续审计和日志排查。

- [x] P003: 在快照数据中暴露耗时信息 <!-- scope: src/loopService.js,src/renderer/types.ts -->
  - 验收要点：`snapshot().tasks` 返回每个任务的开始时间、结束时间、累计耗时毫秒和运行中状态所需字段。
  - 验收要点：运行中任务的耗时展示不依赖额外 IPC 接口，现有定时 `emitUpdate` 可驱动刷新。
  - 验收要点：快照查询仍按 Plan 创建时间和任务顺序排序，新增字段不破坏现有前端消费逻辑。

- [x] P004: 展示任务耗时与计划汇总 <!-- scope: src/renderer/utils/time.ts,src/renderer/components/PlanLists.tsx,src/renderer/pages/WorkspacePage.tsx,src/styles.css -->
  - 验收要点：任务列表展示每个任务的耗时，未开始显示“未开始”，运行中显示“已运行 xx”，已完成/失败/停止显示累计耗时。
  - 验收要点：新增耗时格式化函数，支持毫秒输入并输出秒、分、小时级中文短格式，空值和异常值有兜底显示。
  - 验收要点：任务与计划区域展示当前 Plan 的总耗时/已完成任务耗时汇总，页面在窄屏和长任务标题下不溢出。
  - 验收要点：运行中任务耗时能随现有快照刷新更新，不需要用户手动刷新页面。

- [x] P005: 补充耗时统计回归测试 <!-- scope: scripts/smoke-test.js -->
  - 验收要点：smoke 测试覆盖任务开始后写入 `started_at`，完成后写入 `finished_at` 和正数 `duration_ms`。
  - 验收要点：smoke 测试覆盖失败或停止回退到 `pending` 时耗时仍被累计，后续重试继续累加。
  - 验收要点：smoke 测试覆盖 `snapshot().tasks` 中耗时字段存在且类型符合前端预期。
  - 验收要点：既有 Plan 解析、scope 并发选择、Plan guard、验收完成等断言保持通过。

- [x] P006: 执行构建与回归验证 <!-- scope: package.json,scripts/smoke-test.js,src/database.js,src/loopService.js,src/renderer/types.ts,src/renderer/components/PlanLists.tsx,src/renderer/pages/WorkspacePage.tsx,src/renderer/utils/time.ts,src/styles.css -->
  - 验收要点：`npm run check` 通过，包含 TypeScript、Node 语法检查和脚本检查。
  - 验收要点：`npm run smoke` 通过，确认数据库迁移、任务执行、耗时统计和原有任务系统行为无回归。
  - 验收要点：如验证失败，仅修复与耗时统计直接相关的问题，不扩大修改范围。

## 总体验收标准

- 数据可靠：任务耗时以毫秒为基础存储，开始、完成、失败、停止和重试场景均能得到可解释的累计耗时。
- 兼容安全：旧数据库自动补齐新增字段，历史任务不因空耗时字段导致页面或任务流程异常。
- 展示清晰：任务列表能直观看到每个任务的耗时状态，Plan 区域能看到汇总耗时，运行中耗时会自动刷新。
- 流程无回归：Plan 草稿确认、任务同步、scope 并发执行、手动执行/停止、Plan guard、验收流程保持原有行为。
- 验证通过：完成实现后 `npm run check` 与 `npm run smoke` 均通过。

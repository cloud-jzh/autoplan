# 需求 #50 开发计划：计划卡片支持 popup 菜单停止/删除

## 需求概述

计划卡片需要提供一个 popup 操作菜单，菜单项包含“停止”和“删除”。结合当前项目实现，计划卡片位于工作区“计划与任务”页左侧 Plan 列，已有“并发执行、阅读全文、查看任务、执行草稿”等按钮，但缺少计划级停止和删除入口。

本计划只描述开发方案与验收标准，不修改业务代码。

## 现状分析

当前计划卡片主体在 `src/renderer/components/plans/PlanList.tsx`，包装层 `src/renderer/components/PlanLists.tsx` 负责状态筛选、草稿执行和“查看任务”控制。工作区页面 `src/renderer/pages/WorkspacePage.tsx` 将计划列表与 `runTaskBatches`、`runTask`、计划阅读器、任务列表的 `stopTask` 连接起来。

后端已有任务级停止能力：`src/main.js` 暴露 `tasks:stop`，`src/preload.js` 暴露 `stopTask`，`LoopService.stopTask` 复用 `src/loop/runtime.js` 的运行时停止逻辑。计划级能力目前只存在内部生命周期函数：`LoopService.stopPlanOperations` 能终止某个 plan 的运行时操作，`LoopService.interruptPlan` 能把 plan 标记为 `interrupted` 并挂起未完成任务；删除需求/反馈时，`src/loop/intakeDeletion.js` 已有级联删除关联 plan、plan_tasks、scan_files 和安全删除 plan 文件的参考实现。但主进程、preload 和渲染层尚无 `plans:stop` / `plans:delete` API。

UI 侧已有 popup 菜单样式和交互范式可参考：侧栏对话列表使用 `more-horizontal` 图标、`role="menu"` / `role="menuitem"` 结构和 danger 菜单项；项目卡片样式里已有 `.ctx-menu` 类。计划卡片样式集中在 `src/renderer/styles/components.css` 的 `.plan-*` 规则中，需要新增计划卡片专用菜单样式，避免破坏现有卡片布局与点击选中行为。

## 方案设计

停止计划：新增计划级 stop API，只允许对运行中计划执行。运行中的判断以 plan 状态为 `running` 或存在运行中任务为准。执行时校验 plan 属于当前 project，调用 `stopPlanOperations` 终止该 plan 的活跃操作，并将未完成任务挂起、plan 状态落到 `interrupted`，防止循环立刻重新选择同一计划继续执行。该语义与现有 `interruptPlan` 保持一致，但通过清晰的 `stopPlan` API 暴露给计划卡片。

删除计划：新增计划级 delete API，校验 plan 属于当前 project，先停止该 plan 的活跃操作，再删除 `plan_tasks`、`plans`、对应 `scan_files` 缓存，并安全删除位于当前项目 `docs/plan` 内的 plan markdown 文件。删除计划时不删除需求/反馈本体，但需要清理 `requirements.linked_plan_id` 和 `feedback.linked_plan_id`，避免计划删除后留下不可预览的坏引用；同时记录事件并返回最新 snapshot。

前端菜单：在计划卡片右侧操作区增加更多菜单按钮，使用现有 `Icon` 的 `more-horizontal`、`stop`、`trash` 图标。菜单点击必须被 `isPlanCardInteractiveEvent` 识别为交互元素，不能触发卡片选中。停止项在非运行计划上禁用并提供 title 原因；删除项为危险操作，点击后弹出明确确认，说明会停止运行、删除计划文件和任务，但不会删除需求/反馈记录。操作成功后刷新 snapshot；若当前打开的计划阅读器或选中的计划被删除，现有控制器和页面状态需要随返回 snapshot 清理。

## 总体验收标准

最终验收命令为：`npm test`、`npm run check`、`npm run build`。

验收范围覆盖 `src/loopService.js`、`src/loop/planLifecycle.js`、`src/loop/workspaceFiles.js`、`src/main.js`、`src/preload.js`、`src/renderer/types.ts`、`src/renderer/hooks/useWorkspaceController.ts`、`src/renderer/pages/WorkspacePage.tsx`、`src/renderer/components/PlanLists.tsx`、`src/renderer/components/plans/PlanList.tsx`、`src/renderer/styles/components.css` 以及新增/更新的测试文件。

通过标准：

1. 后端能正确停止运行中的计划并阻止其继续执行，状态与事件可追踪。
2. 后端能安全删除计划、任务、扫描缓存和 docs/plan 内对应文件，同时不删除需求/反馈本体并清理 linked_plan_id。
3. 计划卡片 popup 菜单包含停止/删除，交互可访问，不干扰卡片选中和既有按钮。
4. 删除和停止后的 snapshot 能让计划列表、任务列表、计划阅读器和选中状态同步更新。
5. `npm test`、`npm run check`、`npm run build` 全部通过，无新增类型错误、构建错误或文件长度违规。

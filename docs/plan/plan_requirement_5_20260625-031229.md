# 需求 #5 开发计划：近期事件显示任务执行动态

## 背景

当前系统已有 `events` 表、`snapshot().events` 快照和前端“近期事件/事件流”展示，但任务执行相关事件还不够直观，且缺少明确的“任务开始”动态。需求 #5 要求近期事件能够显示任务执行事件，例如“开始了 xx 任务”“结束了 xx 任务”等，让用户在概览页即可看到任务生命周期变化。

## 任务拆解

- [x] P001: 统一任务事件数据约定 <!-- scope: src/loopService.js,src/renderer/types.ts -->
  - 验收要点：明确任务事件类型与展示语义，至少覆盖任务开始、成功结束、失败结束、停止请求和中断停止场景。
  - 验收要点：任务事件 `meta` 包含 `taskId`、`taskKey`、`taskTitle`、`planId`、`status`、`startedAt`、`finishedAt` 等可选字段，优先复用现有 `events.meta`，不新增数据库表。
  - 验收要点：前端 `AppEvent` 类型能表达结构化任务事件元信息，旧事件 `meta` 为空、非 JSON 或缺少任务字段时仍可安全展示。

- [x] P002: 在任务生命周期写入事件 <!-- scope: src/loopService.js -->
  - 验收要点：任务进入执行前写入“开始了 xx 任务”事件，自动循环、手动执行和并发执行路径均覆盖且不重复记录。
  - 验收要点：任务执行成功后写入“结束了 xx 任务”事件，并保留日志路径、退出码、任务标识等排查所需元信息。
  - 验收要点：任务失败、手动停止、循环停止和 Plan 中断时写入对应失败/停止事件，能从近期事件区看出任务未正常结束。
  - 验收要点：新增事件不改变既有任务状态流转、耗时统计、Plan checkbox 回写和进度区写入逻辑。

- [x] P003: 保证近期事件快照实时可用 <!-- scope: src/loopService.js,src/renderer/types.ts -->
  - 验收要点：`snapshot().events` 按最新优先返回任务开始和结束事件，现有 `ORDER BY id DESC LIMIT 80` 语义不被破坏。
  - 验收要点：任务事件写入后立即触发 `emitUpdate`，概览页“近期事件”和“事件流”页无需手动刷新即可看到变化。
  - 验收要点：并发任务同时开始/结束时，每个任务均有独立事件，事件元信息可区分对应 `taskId` 和 `taskKey`。

- [x] P004: 优化近期事件展示文案 <!-- scope: src/renderer/components/PlanLists.tsx,src/renderer/pages/WorkspacePage.tsx,src/renderer/utils/time.ts,src/styles.css -->
  - 验收要点：“近期事件”和“事件流”统一使用中文任务动态文案，例如“开始了 P001 任务：xxx”“结束了 P001 任务：xxx”。
  - 验收要点：任务开始、成功、失败、停止使用易区分的状态标签或视觉样式，非任务事件仍保持原有可读展示。
  - 验收要点：事件时间继续使用中国时区格式展示，长任务标题、缺失标题和窄屏布局下不溢出、不遮挡操作入口。
  - 验收要点：旧数据中的 `task.executed`、`task.failed` 等事件在没有新 `meta` 时仍能以原始 `message` 兜底显示。

- [x] P005: 补充任务事件回归测试 <!-- scope: scripts/smoke-test.js -->
  - 验收要点：smoke 测试覆盖模拟任务执行成功后，事件列表包含开始事件和成功结束事件，且顺序为最新事件在前。
  - 验收要点：smoke 测试覆盖任务执行失败时写入失败事件，事件 `meta` 中包含任务标识和日志/错误信息。
  - 验收要点：smoke 测试覆盖手动停止或中断路径会产生停止类任务事件，不影响后续任务重试。
  - 验收要点：既有项目、需求、Plan 草稿、任务同步、scope 并发、验收完成等 smoke 断言保持通过。

- [x] P006: 执行构建与回归验证 <!-- scope: package.json,scripts/smoke-test.js,src/loopService.js,src/renderer/types.ts,src/renderer/components/PlanLists.tsx,src/renderer/pages/WorkspacePage.tsx,src/renderer/utils/time.ts,src/styles.css -->
  - 验收要点：`npm run check` 通过，确认 TypeScript、主进程脚本和 smoke 脚本语法均无错误。
  - 验收要点：`npm run smoke` 通过，确认任务事件写入、快照返回和既有核心流程无回归。
  - 验收要点：如验证失败，仅修复与任务事件展示直接相关的问题，不扩大业务修改范围。

## 总体验收标准

- 事件完整：任务开始、成功结束、失败、停止和中断场景均能在事件系统中留下可读记录。
- 展示直观：概览页“近期事件”能直接显示“开始了 xx 任务”“结束了 xx 任务”等中文任务动态，而不是只显示原始事件类型。
- 实时更新：任务事件写入后通过现有快照订阅自动刷新，用户无需切换页面或手动刷新。
- 兼容安全：不破坏旧事件数据、现有 `events` 表结构、任务状态流转、耗时统计和 Plan 进度写回。
- 验证通过：完成实现后 `npm run check` 与 `npm run smoke` 均通过。

# 需求 #63 开发计划：终端没有创建按钮

## 需求描述

终端（Terminal）模块当前在工作区的终端页签下仅展示已有终端会话的列表，缺少新建终端的入口按钮。

## 现状分析

1. **`src/renderer/pages/WorkspacePage.tsx`** 第 637-646 行，终端页签渲染内联组件 `WorkspaceTerminalMetadataSection`（第 737-782 行），该组件只展示会话列表和刷新按钮，无「创建终端」按钮。
2. **`src/renderer/components/workspace/WorkspaceTerminalView.tsx`** 是完整的 xterm.js 终端模拟器组件，包含多个创建按钮（标签栏 `+` 按钮、空态「新建终端」按钮、控制栏「新建」按钮），但该组件未被 `WorkspacePage.tsx` 导入或使用。
3. **preload.js** 已暴露 `window.autoplan.createTerminal()` IPC 接口，后端 `terminalIpc.js` + `terminalService.js` 已实现会话创建流程，无需后端改动。
4. **`TerminalCreateInput`** 类型（`types.ts` 第 1414 行）结构：`{ projectId, cwd?, profileId?, title?, cols?, rows?, env? }`，最小调用只需 `{ projectId }`。
5. 侧栏终端导航已存在且支持徽标计数（`WorkspaceSidebar.tsx` 第 289-290 行），无需改动。

## 影响范围

- `src/renderer/pages/WorkspacePage.tsx`：为 `WorkspaceTerminalMetadataSection` 添加创建按钮、创建状态管理以及终端列表的即时更新逻辑
- `src/renderer/hooks/useTerminalSessions.ts`：无直接改动，但最终可考虑复用其 `createSession` 方法保持一致性

# 开发计划：执行器布局优化（插件式执行器）

## 需求概述

优化执行器系统的交互布局，将其从当前"一次性卡片命令"模式升级为类似 VS Code 插件式的开发工具面板。核心目标：
- 支持类似 `flutter run` 的**编译启动 / 热刷新 / 停止**三态生命周期
- 将卡片网格布局优化为可展开的紧凑列表 + 内联输出面板
- 新增 `plugin` 执行器类型，支持持久进程管理（启动后保持运行，可通过命令/信号触发热刷新）
- 保持与现有 `shell` / `process` 类型完全向后兼容

## 需求分析

### 当前架构理解

| 模块 | 文件 | 职责 |
|------|------|------|
| 配置校验 | `src/executors/executorConfig.js` | 校验 shell/process 两种类型，解析 tasks.json |
| 持久化 | `src/executors/executorStore.js` | CRUD + 运行状态回写 |
| 运行引擎 | `src/executors/executorRunner.js` | 依赖调度 + 一次性 shell 执行 |
| 运行时管理 | `src/loop/runtime.js` | 管理活跃操作，支持 stopExecutor 终止 |
| IPC 通道 | `src/preload.js` / `src/main.js` / `src/loopService.js` | 前后端通信 |
| UI 视图 | `src/renderer/components/workspace/WorkspaceExecutorsView.tsx` | 卡片网格 + 筛选/搜索/操作 |
| UI 编辑器 | `src/renderer/components/workspace/ExecutorEditorModal.tsx` | 弹窗式配置表单 |
| UI 辅助 | `src/renderer/components/workspace/executorEditor.helpers.ts` | 格式化、状态读取 |
| 类型定义 | `src/renderer/types.ts` | TypeScript 类型 |
| 样式 | `src/renderer/styles/workspace.css` | 卡片、网格、状态灯 |

### 待改进点

1. **执行器只有一次性运行模式**：`runExecutor` 启动命令，等待退出，返回结果。无法表示"启动后持续运行、中途可发送热刷新"的持久进程。
2. **UI 布局为粗粒度卡片**：每个执行器占一个固定尺寸卡片，信息密度低，无法在列表内直接查看输出。
3. **缺少插件的概念**：没有 `plugin` 类型，无法表达 Flutter、React Native、Vite dev server 等"长期运行 + 可交互"的开发工具。
4. **运行状态展示不够突出**：运行中的执行器仅通过小 LED 灯显示，状态指示不够直观。

### 设计决策

1. **新增 `plugin` 执行器类型**：`plugin` 类型支持三态生命周期（start → running → stop），并支持 `reload` 子命令（发送给正在运行的进程）。
2. **actions 配置块**：plugin 执行器新增 `actions` 字段，包含 `start`、`reload`、`stop` 三个子命令。`start` 定义启动命令；`reload` 定义热刷新命令（可以是发送给进程 stdin 的文本，也可以是独立命令）；`stop` 定义停止命令（默认发送 SIGTERM）。
3. **进程持久化管理**：在 runtime 层记录 plugin 执行器的子进程 PID，支持向运行中的进程发送 stdin 输入。
4. **UI 布局改为可展开列表**：用紧凑的列表行代替卡片网格，每行展示执行器名称、分组、当前状态、关键操作按钮；点击展开可查看实时输出日志。
5. **完全向后兼容**：现有 `shell` / `process` 执行器行为不变。

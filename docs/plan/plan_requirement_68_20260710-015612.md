# 需求与反馈条目支持人工验收状态

## 需求概要

在需求、反馈记录完成后，于条目下方显示独立的人工验收状态，并提供标记验收与取消验收能力。实现应复用现有 accepted_at 正交状态思路，但目标对象扩展到 requirements 与 feedback，不改变原有执行状态、计划/任务验收流程和计划生成逻辑。

## 任务拆解

- [x] P001: 补齐需求和反馈的验收数据模型 <!-- scope: src/database.js,src/loop/snapshots.js,src/renderer/types.ts -->
  - 验收要点：requirements 与 feedback 表迁移出 accepted_at 字段，新库和旧库升级后都可保存验收时间
  - 验收要点：项目快照中的需求和反馈记录包含 accepted_at，前端类型能直接读取该字段
  - 验收要点：accepted_at 与 status 保持独立，标记验收不要求把 status 改写为 accepted
- [x] P002: 新增需求和反馈验收后端能力 <!-- scope: src/loop/acceptance.js,src/loopService.js,src/main.js,src/preload.js,src/renderer/types.ts -->
  - 验收要点：支持对当前项目内的 requirement 与 feedback 单条标记验收，写入非空 ISO accepted_at 并记录事件
  - 验收要点：支持对已验收或未验收的 requirement 与 feedback 取消验收，清空 accepted_at 并记录事件
  - 验收要点：非法 id、跨项目对象、非法 intake 类型返回明确错误，不写入错误目标
  - 验收要点：需求/反馈验收不触发计划验收命令、不执行任务、不影响现有 plan/task 验收接口
- [x] P003: 在工作区控制器接入需求反馈验收操作 <!-- scope: src/renderer/hooks/useWorkspaceController.ts,src/renderer/pages/WorkspacePage.tsx,src/renderer/types.ts -->

  - 验收要点：验收操作失败时沿用现有错误提示与 snapshot 不更新逻辑
  - 验收要点：现有创建、编辑、删除、中断、恢复、追加任务、重试生成计划操作保持原行为
- [x] P004: 在需求和反馈条目下方展示验收状态与操作 <!-- scope: src/renderer/components/IntakePanel.tsx,src/renderer/styles/components.css -->

  - 验收要点：按钮点击后调用对应处理函数，加载中或重复点击不会造成布局抖动或文本溢出

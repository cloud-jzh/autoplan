# 需求 #6 开发计划：系统界面图标统一使用 Icon

## 背景

当前界面中部分导航、统计卡片、操作按钮、状态节点和输入区按钮直接使用字符符号或 emoji 作为图标。需求 #6 要求系统界面的图标统一使用 icon，而不是使用 emoji，以保证视觉风格一致、跨平台渲染稳定、可访问性明确，并便于后续维护主题色、尺寸和悬浮态。

## 任务拆解

- [x] P001: 建立统一 SVG 图标体系 <!-- scope: src/renderer/components/icons.tsx,src/styles.css -->
  - 验收要点：新增本地图标组件或图标映射，默认使用 SVG icon，不新增非必要运行时依赖。
  - 验收要点：图标组件支持统一的 `size`、`className`、`title` 或 `aria-hidden` 语义，装饰图标不被屏幕阅读器重复朗读。
  - 验收要点：覆盖当前界面需要的概览、需求、反馈、任务、事件、计划、运行、完成、编辑、删除、警告、刷新、附件、发送等基础图标。
  - 验收要点：图标颜色默认继承文本色或使用现有 CSS 变量，能够适配品牌色、成功、警告、危险和普通按钮状态。

- [x] P002: 替换项目页操作图标 <!-- scope: src/renderer/pages/ProjectsPage.tsx,src/styles.css -->
  - 验收要点：项目卡片右键菜单或更多菜单中的进入工作区、编辑、删除操作均使用 SVG icon，按钮文字和点击行为保持不变。
  - 验收要点：删除确认弹窗的警告视觉改为统一 Icon 组件，保留现有危险色、圆形背景和居中布局。
  - 验收要点：项目页不再把 emoji 或字符符号作为操作图标来源，搜索、创建、编辑、删除流程无回归。
  - 验收要点：图标与文字间距、对齐、悬浮态和危险态在窄屏及长项目名称下不挤压、不错位。

- [x] P003: 替换工作区导航与统计图标 <!-- scope: src/renderer/pages/WorkspacePage.tsx,src/styles.css -->
  - 验收要点：工作区侧边导航的概览、需求、反馈、任务与计划、事件流图标改为统一 SVG icon，当前选中态样式保持一致。
  - 验收要点：概览统计卡片中的需求数、计划数、任务进度、循环间隔图标改为 Icon 组件，`StatCard` 不再依赖字符串图标。
  - 验收要点：图标替换后统计卡片的颜色、尺寸、圆形背景、文字层级和响应式布局与现有视觉保持一致或更稳定。
  - 验收要点：工作区标题、返回项目列表、查看任务等文字入口不因图标替换丢失可点击区域或可访问标签。

- [x] P004: 替换流水线与输入区图标 <!-- scope: src/renderer/pages/WorkspacePage.tsx,src/renderer/components/Composer.tsx,src/styles.css -->
  - 验收要点：循环阶段流水线中的完成态和运行态使用 SVG icon，未开始步骤仍可显示序号，阶段状态语义保持清晰。
  - 验收要点：需求/反馈输入区的添加附件、发送按钮和非图片附件占位改为 Icon 组件，不再使用字符图形充当图标。
  - 验收要点：按钮 `aria-label`、键盘触发、文件选择、拖拽上传和提交表单行为保持不变。
  - 验收要点：输入区图标在禁用、悬浮、拖拽高亮和紧凑布局下尺寸统一，不影响文本区域高度自适应。

- [x] P005: 同步静态界面原型图标 <!-- scope: ui/projects.html,ui/workspace.html,ui/README.md -->
  - 验收要点：静态项目页和工作区原型中用于导航、统计、操作、警告、附件和流水线状态的 emoji 或字符符号改为 inline SVG 或可复用 icon 片段。
  - 验收要点：原型页面的视觉说明与实际 React 界面保持一致，`ui/README.md` 补充“界面图标使用 icon，不使用 emoji”的设计约束。
  - 验收要点：原型中的示例业务文案、时间、状态、占位内容不被误删，仅替换承担图标职责的符号。
  - 验收要点：浏览器直接打开静态 HTML 时图标可见，无需依赖外部 CDN 或网络资源。

- [x] P006: 执行图标替换回归验证 <!-- scope: package.json,src/renderer/components/icons.tsx,src/renderer/pages/ProjectsPage.tsx,src/renderer/pages/WorkspacePage.tsx,src/renderer/components/Composer.tsx,src/styles.css,ui/projects.html,ui/workspace.html -->
  - 验收要点：`npm run check` 通过，确认 TypeScript、主进程脚本和现有检查无错误。
  - 验收要点：`npm run build` 通过，确认新增或替换的 SVG icon 可被 Vite 正常构建。
  - 验收要点：对 `src/renderer`、`src/styles.css` 和 `ui` 执行字符扫描，确认无承担图标职责的 emoji 或字符符号残留。
  - 验收要点：手动核对项目页、工作区概览、任务与计划、事件流、需求/反馈输入区，确认图标展示清晰且主要交互无回归。

## 总体验收标准

- 图标统一：系统界面中所有承担图标职责的元素均使用 SVG icon 或统一 Icon 组件，不再使用 emoji 作为图标。
- 视觉一致：图标尺寸、线宽、颜色、悬浮态、危险态和选中态遵循现有设计变量，页面整体风格统一。
- 交互无回归：项目管理、工作区导航、输入提交、附件上传、任务查看、删除确认等核心交互保持原有行为。
- 可访问性明确：装饰性图标正确隐藏，功能性按钮保留可读 `aria-label` 或文字说明，键盘操作不受影响。
- 构建通过：完成实现后 `npm run check` 与 `npm run build` 均通过，静态原型无需外部资源即可正常显示图标。

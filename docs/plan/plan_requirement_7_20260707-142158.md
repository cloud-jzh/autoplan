# 计划生成耗时记录

## 需求概要

需求 #7 要求“计划生成，也需要记录耗时”。当前项目已在任务执行、脚本、执行器链路记录 duration_ms/last_duration_ms，但计划生成链路只在 plans 表记录 created_at/updated_at，并通过 plan.generated / plan.generate.failed 事件记录结果，缺少从开始生成到成功落盘或失败的耗时。实现应覆盖三类计划生成策略（external-cli-markdown、external-cli-structured、builtin-llm-structured）以及普通需求/反馈、目录扫描、分阶段计划生成路径；在成功时把生成耗时持久化到 plans，在成功/失败事件 meta 中写入 durationMs，快照和前端计划列表展示可读的生成耗时，并补充单测确保成功、失败、结构化和分阶段场景都能追踪。

## 任务拆解

- [x] P001: 为 plans 持久化计划生成耗时字段 <!-- scope: src/database.js,src/loop/planLifecycle.js -->
  - 验收要点：plans 表新增 plan_generation_duration_ms 整数列，默认值为 0，并通过 ensureColumn 兼容既有数据库升级。
  - 验收要点：insertPlan 支持接收 planGenerationDurationMs 或同等字段，并仅在 plans 表存在该列时写入非负整数耗时。
  - 验收要点：未提供耗时时保持旧行为，旧数据和旧测试夹具不会因为缺少该字段而失败。
- [x] P002: 在所有计划生成成功路径计算并写入耗时 <!-- scope: src/loop/planGeneration.js -->
  - 验收要点：external-cli-markdown 的目录扫描生成、需求生成、反馈生成成功后，plans.plan_generation_duration_ms 记录从进入该次生成函数到计划成功插入数据库的耗时。
  - 验收要点：external-cli-structured 与 builtin-llm-structured 生成成功后同样写入生成耗时，包含 stdout 兜底恢复、PlanSpec 解析、渲染和入库时间。
  - 验收要点：分阶段计划生成成功时，每个阶段 plan 都写入合理的生成耗时；可以记录整次分阶段生成耗时，也可以按阶段插入时共享本次总耗时，但必须保证每条生成出的 plans 记录非空、非负。
  - 验收要点：耗时单位统一为毫秒，运行时计算使用单调递增时间源或 Date.now 差值，并通过归一化避免负数、NaN 或空字符串落库。
- [x] P003: 在计划生成事件中携带耗时元数据 <!-- scope: src/loop/planGeneration.js,src/loopService.js,src/loop/taskEvents.js -->
  - 验收要点：plan.generated 事件 meta 增加 durationMs 或 planGenerationDurationMs 字段，值与对应 plans 记录的 plan_generation_duration_ms 一致或语义一致。
  - 验收要点：plan.generate.failed、结构化解析失败、内置 LLM 生成失败、缺失 plan 文件等失败事件 meta 同样携带本次生成耗时，便于从事件流回溯失败耗时。
  - 验收要点：失败回写到 requirements/feedback 的 generate_fail_count、last_generate_* 逻辑保持不变，仅新增耗时元数据，不改变重试节流语义。
  - 验收要点：事件 meta 中的耗时字段名称在成功和失败路径保持一致，方便前端与日志检索复用。
- [x] P004: 通过快照和前端类型暴露计划生成耗时 <!-- scope: src/loop/snapshots.js,src/renderer/types.ts -->
  - 验收要点：snapshot 返回的 Plan 对象包含 plan_generation_duration_ms，缺失或非法值归一化为 0。
  - 验收要点：renderer 的 Plan 类型增加 plan_generation_duration_ms 字段，保持与后端 snake_case 字段一致。
  - 验收要点：现有 planBackendSnapshotFields、Claude token 脱敏、并发建议和标题读取逻辑不受新增字段影响。
- [x] P005: 在计划列表展示生成耗时 <!-- scope: src/renderer/components/plans/PlanList.tsx,src/renderer/utils/planTasks.ts,src/renderer/utils/time.ts -->
  - 验收要点：计划卡片在现有任务耗时摘要之外，展示“生成耗时”或等价文案，使用项目既有 formatDuration 风格格式化毫秒值。
  - 验收要点：plan_generation_duration_ms 为 0、缺失或旧数据时显示为未记录/0 秒等明确兜底，不造成布局错位。
  - 验收要点：正在运行任务的任务耗时动态展示逻辑保持不变，生成耗时不参与任务总耗时和已完成任务耗时统计。
  - 验收要点：计划列表空状态、草稿计划、已完成计划、中断/停止计划的卡片布局仍可正常渲染。
- [x] P006: 补充计划生成耗时单测 <!-- scope: src/loop/planGeneration.test.js,src/loop/snapshots.test.js,src/renderer/utils/planTasks.test.ts -->
  - 验收要点：覆盖普通 Markdown 计划生成成功时 insertPlan 收到非负 planGenerationDurationMs，plan.generated 事件 meta 带耗时。
  - 验收要点：覆盖结构化 PlanSpec 或内置结构化生成成功时同样记录耗时。
  - 验收要点：覆盖计划生成失败路径会在失败事件 meta 中携带耗时，且不插入 plans 记录。
  - 验收要点：覆盖 snapshot 对 plan_generation_duration_ms 的归一化输出和前端格式化函数对 0、正数、非法值的兜底。
- [x] P007: 完整验收 <!-- scope: validation -->
  - 验收要点：最终验收命令为 npm test && npm run build
  - 验收要点：全量 Node 单测通过，新增耗时字段不破坏既有计划生成、任务同步、失败重试、快照和前端工具函数测试。
  - 验收要点：TypeScript 与 Vite 构建通过，Plan 类型、计划列表展示和新增字段引用无类型错误。
  - 验收要点：实际生成成功的 plan 在 plans 表、snapshot.plans、plan.generated 事件 meta 中均可看到非负生成耗时。
  - 验收要点：计划生成失败时不创建 plan，但对应失败事件 meta 中能看到本次生成耗时。

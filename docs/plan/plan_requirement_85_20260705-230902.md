# 检测到正式版更新后自动下载安装包并支持打开

## 需求概要

当前 Electron 应用已有基于 GitHub Releases 的正式版更新检查、全局更新横幅和设置页入口，但只提醒用户前往 Release 页面手动下载。需求 #85 要求在检测到可用更新后自动选择当前平台安装包下载到本地，并在横幅/设置页提供打开安装包的操作。实现应复用现有 updateChecker、主进程 IPC、preload API 和 React 更新状态链路，不引入自动安装行为。

## 任务拆解

- [x] P001: 扩展 Release 解析以识别当前平台安装包资产 <!-- scope: src/updateChecker.js,src/updateChecker.test.js -->
  - 验收要点：parseLatestRelease 能从 GitHub Release assets 中提取可下载安装包的名称、下载地址、大小、平台和架构等元数据，并继续保留现有版本、正式版、摘要字段。
  - 验收要点：资产选择规则能优先匹配当前运行平台和架构：Windows 选择 NSIS Setup .exe，macOS 选择 dmg/zip，Linux 选择 AppImage 或 deb；找不到匹配资产时状态中明确体现无可用安装包。
  - 验收要点：prerelease、draft、非法版本和无 assets 的 Release 不触发可下载安装包状态，现有稳定版更新判断不回归。
- [x] P002: 实现检测到可用更新后的安装包自动下载状态机 <!-- scope: src/updateChecker.js,src/main.js,src/database.js -->
  - 验收要点：手动检查和定时自动检查在发现未忽略的正式版更新且存在匹配安装包资产后，会自动开始下载，不需要用户再点击下载按钮。
  - 验收要点：下载文件保存到应用 userData 下的受控更新下载目录，并按版本或资产名称隔离，避免覆盖非本应用文件。
  - 验收要点：更新状态包含下载阶段、进度、错误信息、本地安装包路径、资产信息和完成时间；下载完成后这些状态可被后续 updateStatus 读取。
  - 验收要点：并发检查或重复状态推送不会启动多个相同安装包下载；已下载且文件仍存在时不会重复下载。
  - 验收要点：网络失败、HTTP 非 2xx、写入失败、用户已忽略版本、无匹配资产等场景都返回结构化状态，不影响应用启动和其他功能。
- [x] P003: 新增打开已下载安装包的主进程 IPC 与 preload API <!-- scope: src/main.js,src/preload.js,src/renderer/types.ts -->
  - 验收要点：渲染进程可通过受控 API 请求打开当前更新安装包，主进程只允许打开已记录且位于应用更新下载目录内的本地文件。
  - 验收要点：安装包存在时使用系统方式打开或在文件管理器中定位，返回 ok/error 结构化结果；文件缺失、路径越界、尚未下载完成时返回明确错误。
  - 验收要点：原有 openExternal 仍仅用于安全打开 http/https 外链，不被扩展为打开本地安装包路径。
- [x] P004: 更新全局横幅和关于页的安装包下载/打开交互 <!-- scope: src/renderer/hooks/useUpdateStatus.ts,src/renderer/components/UpdateNotice.tsx,src/renderer/components/workspace/WorkspaceSettingsView.tsx,src/renderer/styles/components.css,src/renderer/styles/workspace.css -->
  - 验收要点：检测到更新后横幅展示自动下载状态：等待下载、下载中进度、下载失败、已下载，并在已下载时提供“打开安装包”按钮。
  - 验收要点：关于/更新设置页展示安装包资产名称、大小、本地状态、错误信息和打开入口，同时保留立即检查、自动检查开关和打开 GitHub Releases 的兜底入口。
  - 验收要点：下载失败或无匹配安装包时，UI 不阻塞用户继续使用，并提供打开 Release 页面手动下载的替代路径。
  - 验收要点：点击“稍后提醒”后本轮不再显示更新横幅，也不会继续为被忽略版本重复启动自动下载。
- [x] P005: 补齐更新下载链路的回归测试 <!-- scope: src/updateChecker.test.js,src/renderer/components/shared.test.tsx,src/renderer/types.syntax.test.js -->
  - 验收要点：单元测试覆盖 Release assets 解析、平台安装包选择、无匹配资产、prerelease/draft 不下载等场景。
  - 验收要点：更新检查状态测试覆盖可用更新自动进入下载状态、下载完成复用本地文件、下载失败保留错误和并发去重。
  - 验收要点：渲染侧或类型测试覆盖新增 UpdateStatus 字段、open installer API 类型，以及横幅在下载中、失败、已下载时的关键按钮文案和状态。

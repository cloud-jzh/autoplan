# 需求 #8 开发计划：增加搜索功能

## 背景

当前 AutoPlan 的项目列表已支持按项目名称、路径和备注进行前端搜索，但进入单个项目工作区后，需求、反馈、计划草稿、Plan、任务和事件流分别分布在不同模块中。随着任务执行和事件积累，用户难以快速定位某条需求、某个任务、某份计划或相关运行记录。

需求 #8 要求“增加搜索功能”。本计划默认将搜索能力落在当前项目工作区内：提供统一搜索入口，按当前项目快照数据进行客户端检索，覆盖需求、反馈、计划草稿、Plan、任务和事件流；保留项目列表现有搜索能力，并在交互和样式上保持一致。

  - 验收要点：搜索模型不改变后端 `snapshot()` 数据结构，不要求数据库新增字段或迁移，避免扩大需求范围。

## 总体验收标准

- 搜索覆盖：当前项目工作区内的需求、反馈、计划草稿、Plan、任务和事件流均可通过统一关键字检索。
- 项目隔离：搜索只作用于当前项目快照，不跨项目展示数据，也不改变数据库内容或后台循环状态。
- 交互清晰：用户能看到搜索输入、命中数量、分组结果、无结果提示和清空入口；空查询时所有列表恢复原始展示。
- 性能稳定：搜索在前端基于现有快照和 memo 化计算完成，不新增高频 IPC、不阻塞任务执行日志刷新。
- 可访问性可用：搜索输入和清空操作支持键盘访问，关键按钮具备可读标签，窄屏布局不溢出。
- 流程无回归：项目管理、需求/反馈提交与编辑、Plan 草稿确认、任务执行/停止、事件流展示和现有项目页搜索保持原有行为。
- 验证通过：完成实现后 `npm run check` 与 `npm run build` 均通过，必要的搜索工具回归校验通过。

# Claude CLI 模型配置支持多配置切换与默认选择

## 需求概要

当前 Claude CLI 的自定义连接（baseUrl / authToken / model）仅作为 plan 生成与执行阶段的单一内联字段存储。需求 #93 要求将其升级为「多配置」：用户可保存多个命名的 Claude CLI 连接配置，通过下拉组件切换选用，并可指定一条默认配置。整体复用既有 ai_configs 多配置范式（全局表 + CRUD 服务 + IPC 广播 + 卡片管理 UI + 快照脱敏 + 默认回退解析）。后端在 provider=claude 时按「所选配置 ID → 默认配置 → 既有内联字段」优先级解析出 baseUrl/authToken/model 注入 buildClaudeEnv，保持对旧内联字段的向后兼容。

# 对话窗口支持 Codex CLI 作为后端（spawn codex exec 替代 HTTP API）

## 需求概要

在现有对话窗口（ChatView → chatController → llmClient 的 HTTP API 直连 + 主进程内 agent loop + chatTools 本地工具）之外，新增 Codex CLI 后端：把对话后端从 HTTP API 换成 spawn `codex exec`，Codex 在工作区里自主完成工具调用。实现纯 Codex 自主执行（不挂 chatTools、跳过 runAgentLoop 的 agent loop、一次 codex exec 完成对话），通过 ai_configs 新增 codex provider 作为配置入口，并复用 Codex session resume 做会话保持（每条对话绑定一个 codex sessionId 存到 conversations 表，后续消息走 codex exec resume）。改动分七部分：1) database 给 conversations 加 codex_session_id 列；2) 新增 codexJsonPrinter 把 codex --json NDJSON 事件翻译成 chatController 现有事件协议；3) 新增 codexChatBackend 驱动 runCodexChat（复用 agentCli 的 codex 参数封装 / spawn / chunk 解码 / session 提取 / resume 失败回退）；4) chatController 按 provider 分流到 runCodexAgentLoop；5) aiConfigService 扩展 codex provider；6) main.js 的 controller 工厂注入 codex 后端启动参数；7) 前端最小改动（types 与配置表单支持选择 codex 并隐藏无关字段，useChat.ts 因事件协议不变而无需改动）。

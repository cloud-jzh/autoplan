# AutoPlan v0.2.1-beta.3

> **Beta 未测试版本**：这是面向快速分发的预发布版本，未经过完整人工测试，不等同于稳定正式版，也不承诺零缺陷。

## 近期更新摘要

本次 beta 累计合入自 v0.2.1-beta.2 以来的多项功能与修复，覆盖 Codex 配置、MCP 接入、计划管理、工作区交互和界面优化。

- **#28 Codex 新增 `xhigh` 思考深度**：在 `low` / `medium` / `high` 之外新增最深入的 `xhigh` 档位，贯穿后端归一化、命令参数拼装、前端表单选项与任务展示，默认值仍为 `medium`，Claude CLI 行为不受影响。
- **#29 本机 MCP 外部接入**：新增 MCP server，外部客户端可通过 `create_project`、`create_requirement`、`create_feedback` 工具创建项目、提交需求和反馈，与 GUI IPC 复用同一套写入逻辑，默认仅本机访问。
- **#30 架构重构与超长文件治理**：拆分 `loopService`、`WorkspacePage`、`PlanLists` 等超大文件为职责清晰的模块（运行时、计划生成、任务执行、快照、并发分析、工作区扫描、页面 Hook 与子视图），并新增文件长度巡检脚本纳入检查链路。
- **#31 计划排序、草稿计划与状态 Tab**：计划列表支持调节执行顺序并持久化，对话创建计划可勾选「草稿」，计划列表新增「全部 / 执行中 / 已完成 / 草稿」Tab，草稿计划默认不参与自动执行。
- **#32 工作区默认进入需求界面**：打开项目工作区且未指定 Tab 时默认显示需求列表与录入区，合法 URL `tab` 参数与手动切换不受影响。
- **#36 任务页与设置页按设计稿优化**：对齐 `ui/tasks.html`、`ui/settings.html` 的双列布局、Plan 卡片、任务分组、设置分组导航、分段控件与响应式细节。
- **#37 计划阅读与 Markdown 预览修复**：计划阅读全文弹窗不再单独展示任务拆解摘要，修复 Markdown 预览中 AutoPlan 任务行 `- [ ] P001: ... <!-- scope: ... -->` 的渲染冲突。
- **#14 修复 macOS 安装包提示损坏**：完善发布流水线 macOS 产物的签名与公证处理逻辑。
- **#16 任务分组折叠与搜索结果弹窗定位**：任务列表分组支持折叠，搜索结果改为弹窗展示并支持点击定位到指定记录。
- **#17 事件提示类型中文化**：将 `scan.done`、`feedback.created`、`plan.generated` 等英文事件类型标题替换为中文。
- **#18 移除搜索框旁的可搜索数量显示**。
- **#19 Plan 列表选择联动任务列表过滤**：在计划列表选中某计划后，任务列表自动过滤为该计划关联任务，并提供重置入口。
- **#20 beta 发布流程补充**：新增自动 beta 发布脚本与发布说明文档。

## Beta 风险提示

本 beta 版本用于更快分发近期修复和功能更新。当前策略是：AI 修复问题后，只要项目编译通过即可发布，以减少人工测试和等待时间。因此，下载测试或日常使用中仍可能遇到功能缺陷、平台差异或安装包问题。本次变更范围较大，涉及工作区重构、MCP 接入、计划管理、任务执行与 UI 多处调整，建议谨慎升级到生产环境。

## 问题反馈

如果你发现问题，请直接到 GitHub Issue 界面提交反馈，并尽量包含：

- 使用的操作系统与安装包类型。
- 复现步骤、预期行为和实际行为。
- 相关截图、日志或附件。

后续 AutoPlan/AI 流程会持续从 Issue 中发现问题，生成修复计划并继续处理。

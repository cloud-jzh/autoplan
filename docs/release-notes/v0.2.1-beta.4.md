# AutoPlan v0.2.1-beta.4

> **Beta 未测试版本**：这是面向快速分发的预发布版本，未经过完整人工测试，不等同于稳定正式版，也不承诺零缺陷。

## 近期更新摘要

本次 beta 聚焦 Agent CLI 后端能力：让 Claude CLI 像 Codex 一样在同一个 plan 内延续会话上下文，并新增 OpenCode CLI 作为第三种可选后端。

- **Claude plan 级会话上下文**：Claude CLI 任务执行会使用计划级 `agent_cli_session_id` 记录和恢复上下文，避免同一 plan 的后续执行、修复和验收重试丢失前序对话。Claude 会话字段与 Codex 的 `codex_session_id` 隔离，不会把 Codex resume 参数或思考深度传给 Claude。
- **新增 OpenCode CLI 后端**：在 Codex 与 Claude 之外新增 OpenCode 作为独立 Agent CLI 后端，贯穿项目级配置、单条需求/反馈覆盖、计划生成快照、任务执行、修复、验收重试、事件 meta 与前端展示。
- **OpenCode 非交互执行与会话复用**：默认命令为 `opencode`，固定使用 `opencode run --format default`；prompt 以位置参数传入，输出从 stdout 写入 last message。同一 plan 首次执行通过 `--title` 创建并记录 session id，后续执行通过 `--session` 恢复。
- **后端隔离与可观测性**：Codex、Claude、OpenCode 各自使用独立会话字段和展示文案；非 Codex 后端不显示、不保存 Codex 思考深度。CLI 缺失、退出码失败、last message 写入失败等场景会给出带后端名称和日志位置的可读错误。
- **外部 MCP 入口同步**：`create_project`、`create_requirement`、`create_feedback` 的 `agentCliProvider` 现在也接受 `opencode`，外部客户端可以直接创建 OpenCode 驱动的项目或单条需求/反馈。
- **回归覆盖扩展**：新增 Claude 会话上下文、OpenCode 后端路由、stdout last message、位置参数 prompt、会话隔离、前端类型与展示、MCP 工具写入等冒烟/单元测试。

## Beta 风险提示

本 beta 版本用于更快分发近期修复和功能更新。当前策略是：AI 修复问题后，只要项目编译通过即可发布，以减少人工测试和等待时间。因此，下载测试或日常使用中仍可能遇到功能缺陷、平台差异或安装包问题。本次变更涉及多个 Agent CLI 后端和会话恢复逻辑，建议在真实 Claude/OpenCode CLI 已安装并完成认证的环境中先做小范围验证。

## 问题反馈

如果你发现问题，请直接到 GitHub Issue 界面提交反馈，并尽量包含：

- 使用的操作系统与安装包类型。
- 选择的 CLI 后端、命令路径和是否已完成 CLI 认证。
- 复现步骤、预期行为和实际行为。
- 相关截图、日志或附件。

后续 AutoPlan/AI 流程会持续从 Issue 中发现问题，生成修复计划并继续处理。

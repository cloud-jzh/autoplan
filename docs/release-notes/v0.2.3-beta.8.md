# AutoPlan v0.2.3-beta.8

> **Beta 未测试版本**：这是面向快速分发的预发布版本，未经过完整人工测试，不等同于稳定正式版，也不承诺零缺陷。

## 近期更新摘要

- 新增需求/反馈引用（@提及）能力：在 Composer 输入 `@需求#1`、`@反馈#2` 时可自动补全当前项目内的候选记录，支持上下方向键导航、回车/Tab 选中、Esc 关闭；需求/反馈正文中已识别的引用会渲染为可点击链接，点击可在需求与反馈面板间定位跳转。
- 计划生成支持引用上下文注入：当需求/反馈正文包含 `@需求#`/`@反馈#` 引用时，会自动把对应记录的标题、状态与摘要注入计划生成 prompt（仅限当前项目内可读取的记录，引用自身时跳过以避免重复）。
- 新增需求/反馈人工验收：需求列表和反馈列表支持对单条记录标记验收（写入 `accepted_at`，不改变执行态 status，不触发计划/任务验收或执行链路），可取消验收。
- 新增「用户指定 skill 传递约束」：当需求/反馈正文明确指定某个 skill、工具或技能调用时，计划生成不会把“使用某 skill/调用某工具”拆成独立任务，而是写入真正需要执行的具体开发任务中，确保执行阶段能识别并使用指定 skill。
- 补充数据库 schema（requirements/feedback 表新增 `accepted_at` 字段及迁移）、IPC/preload 通道、渲染层类型定义与相关测试覆盖，更新冒烟测试。

## Beta 风险提示

本 beta 版本用于更快分发近期修复和功能更新。当前策略是：AI 修复问题后，只要项目编译通过即可发布，以减少人工测试和等待时间。因此，下载测试或日常使用中仍可能遇到功能缺陷、平台差异或安装包问题。

## 问题反馈

如果你发现问题，请直接到 GitHub Issue 界面提交反馈，并尽量包含使用的操作系统、安装包类型、复现步骤、截图或日志。后续 AutoPlan/AI 流程会持续从 Issue 中发现问题，生成修复计划并继续处理。

# AutoPlan v0.2.3-beta.9

> **Beta 未测试版本**：这是面向快速分发的预发布版本，未经过完整人工测试，不等同于稳定正式版，也不承诺零缺陷。

## 近期更新摘要

- 适配 opencode CLI：无人值守执行由 `--dangerously-skip-permissions` 切换为 `--auto`，自动批准未被显式拒绝的工作区访问权限，避免工具调用被自动拒绝导致写不出计划；与 codex 的 `--sandbox danger-full-access`、claude 的 `--dangerously-skip-permissions` 对齐。
- 修复 Windows 下 opencode 多行 prompt 投递：`.cmd/.bat` shim 会把 CR/LF 解释为命令边界，结构化 prompt 只剩首行。多行 prompt 现在无条件落盘为附件，以 `-f` 作为消息附件安全投递；位置消息固定放在 `-f` 之前，避免被解析器吞入 file[]。附件写入失败时直接返回可诊断失败，不再静默降级为会被截断的位置参数。
- codex 会话（新建与续接）追加 `--skip-git-repo-check`，避免在非 git 工作目录下被拦截。
- 计划生成新增确定性「需求描述」注入：无论模型直接写盘还是 stdout 兜底恢复，都会用 intake 原始正文写入计划 `## 需求描述` 章节（逐行引用块形式），确保附件说明、引用上下文、项目 README 不会污染计划正文；Markdown 与结构化（含 builtin-llm）两条生成路径均已覆盖，并补充对应格式约束与测试。

## Beta 风险提示

本 beta 版本用于更快分发近期修复和功能更新。当前策略是：AI 修复问题后，只要项目编译通过即可发布，以减少人工测试和等待时间。因此，下载测试或日常使用中仍可能遇到功能缺陷、平台差异或安装包问题。

## 问题反馈

如果你发现问题，请直接到 GitHub Issue 界面提交反馈，并尽量包含使用的操作系统、安装包类型、复现步骤、截图或日志。后续 AutoPlan/AI 流程会持续从 Issue 中发现问题，生成修复计划并继续处理。

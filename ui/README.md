# AutoPlan UI 设计稿

纯 HTML/CSS 设计稿，浏览器直接打开即可预览，无需依赖。

项目和工作区是**两个独立的界面、各自不同的布局**，分属两个文件：

## 文件

| 文件 | 界面 | 布局 |
|------|------|------|
| `projects.html` | **项目管理** | 顶部 header + 全宽内容区（项目卡片网格 + CRUD） |
| `workspace.html` | **工作区** | 左侧边栏导航 + 主区（概览/需求/反馈/任务/事件） |

## projects.html · 项目管理
- 顶部 sticky header：品牌 + 「打开工作区」入口 + 「创建项目」
- 页面标题区 + 概览统计（项目总数 / 运行中 / 异常）
- 工具栏：搜索框 + 状态分段筛选（全部/运行中/已停止/异常）
- 项目卡片网格：头像、名称、路径、描述、计划/任务/间隔统计、状态指示灯、更新时间
- 卡片右键菜单：打开工作区 / 编辑 / 复制 / 删除
- 完整 CRUD：
  - **创建/编辑**：模态框（图标色 6 选 1、名称*、路径*、验收命令、间隔、描述）
  - **复制**：生成 `-copy`，状态置为已停止
  - **删除**：二次确认，提示不删磁盘文件
- 点击卡片 / 「打开工作区」→ 跳转 `workspace.html`

## workspace.html · 工作区
恢复为第一版的工作区布局（侧边栏导航），五个视图间切换：
- **概览** — 统计卡 + 渐变循环状态卡 + 阶段流水线 + 近期事件
- **需求 / 反馈** — 流式列表 + 悬浮圆角输入框 + 附件
- **任务与计划** — 循环控制 + 计划草稿 + Plan/Task 网格
- **事件流** — 循环运行日志表格
- 侧边栏底部显示当前循环状态

## 设计说明
- 两个界面共享同一套设计 token（品牌色、语义色、圆角、阴影），保持视觉统一。
- 项目页用**顶栏 + 全宽网格**，工作区用**侧边栏 + 分视图**，布局明显区分。
- 状态语义化：指示灯 LED（脉冲动画）、彩色状态芯片、进度条。
- 配色：在原蓝色基调上引入紫蓝渐变 + 中性灰阶层级，浅色主题。
- 界面图标统一使用本地 inline SVG / icon 片段，不使用 emoji 或字符图形充当图标，确保静态 HTML 离线打开时视觉与 React 界面一致。
